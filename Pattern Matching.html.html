Pattern Matching in Python

Pattern Matching was introduced in Python 3.10 as a new feature that allows developers to match complex data structures with ease. This feature is inspired by the switch-case statements found in other programming languages like C and Java, but Pythonâ€™s pattern matching is more flexible and powerful.

Key Benefits of Pattern Matching:

  • Clearer and more readable code for matching patterns in data structures.
  • Supports matching based on both values and data structure shapes.
  • Allows you to break down and extract parts of data easily.

How Pattern Matching Works

Pattern matching uses the match and case keywords. When you use pattern matching, you can define multiple cases that Python will evaluate. Each case can match different patterns, including literals, data structures like lists and dictionaries, and more.

Here's an example of how pattern matching is implemented:


def match_value(value):
    match value:
        case 1:
            return "It's one!"
        case [1, 2, 3]:
            return "It's a list with values 1, 2, 3."
        case _:
            return "It's something else."

Pattern Matching with Complex Data Structures

Pattern matching shines when working with more complex data structures, such as lists, dictionaries, and custom objects. You can match specific values inside a data structure while ignoring others or extracting values into variables.

Here's an example with a dictionary:


def match_dict(data):
    match data:
        case {"name": name, "age": age}:
            return f"Name: {name}, Age: {age}"
        case _:
            return "Unrecognized structure"

When to Use Pattern Matching

Pattern matching is ideal when you have multiple conditions to check for in a complex structure. It is particularly useful in scenarios like:

  • Parsing configuration files or command-line arguments.
  • Dealing with nested or recursive data structures (e.g., JSON or XML).
  • Handling multiple conditions for extracting values from structured data.

By leveraging pattern matching, developers can write more maintainable and readable code, especially in cases where they need to work with structured data.
